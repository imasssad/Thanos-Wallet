# LEP100 v8.1 additions

This branch wires the LEP100 sync job to the live Makalu source configuration:

- RPC: `https://rpc.litho.ai`
- Chain ID: `700777`
- Explorer token index: `https://makalu.litho.ai/tokens`

## What changed
- Added `makalu-lep100-source.ts` as the canonical source config for Makalu.
- Replaced placeholder `lep100:makalu:*` addresses with a hybrid registry model:
  - verified full addresses can be injected with environment variables such as `MAKALU_LEP100_COLLE_ADDRESS`
  - explorer-provided address previews remain visible in the registry for every seeded token
- Updated the indexer sync job so it attempts live metadata reads from Makalu RPC for any token that has a verified full address configured.
- Added source metadata to sync responses so operators can confirm the RPC and explorer endpoints in use.

`LITBTC2` remains excluded.

## Why the registry still uses previews by default
The uploaded Makalu contract-address document surfaces address previews such as `0xE7eBf52b...60DF49`, not full 42-character addresses. That is enough to seed the explorer-linked registry, but not enough for direct RPC reads. To complete live RPC reads per token, provide the final full addresses through environment variables or replace the preview values after verification.
